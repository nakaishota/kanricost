# 管理コスト診断

「管理コスト(物事を維持・管理する手間)をどれだけ嫌うか」を日常のシナリオから診断し、ミニマリスト度を占うエンタメ系Web診断サイト。

**公開サイト → https://nakaishota.github.io/kanricost/**

- **静的SPAで完結**(サーバー・DB・実行時AI不要)
- モノ / デジタル / 人間関係・予定 / お金・契約 の4領域、100問プールから毎回20問をランダム抽選
- 結果は8タイプ分類 + ミニマリスト度% + 領域別内訳 + 細分化された傾向コメント
- Vite + React + TypeScript で構築、診断ロジックは Vitest でテスト

## 構成

```
src/
├── App.tsx              # 画面遷移の状態機械(start | quiz | result)
├── types.ts             # ドメイン型
├── data/
│   ├── domains.ts       # 4領域の定義(名前・コード・アイコン・配点定数)
│   ├── questions.ts     # 問題プール(各領域25問・計100問)
│   ├── resultTypes.ts   # 8タイプの定義(文言・アイコン)
│   └── insights.ts      # 細分化コメント(領域別・スコア帯)
├── logic/
│   ├── random.ts        # Fisher–Yates(rng注入可)
│   ├── selectQuestions.ts # 領域別5問の層化抽出 + シャッフル
│   ├── scoring.ts       # 集計・最大領域・タイプ判定(純関数)
│   ├── share.ts         # シェアテキスト/Xインテント/コピー
│   └── __tests__/       # Vitest(スコア境界値・抽選バランス・データ整合性)
└── components/          # 画面・パーツ
```

## スコアリング

各選択肢に 0 / 2 / 4 / 5 の段階点(管理コスト回避度)を持たせ、20問×最大5点 = **100点満点 = ミニマリスト度%**。領域別(各25点満点)の内訳と偏りからタイプを判定する。判定ロジックは `src/logic/scoring.ts` に純関数として隔離し、テストで担保している。
